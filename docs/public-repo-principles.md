# Public Repo Principles

These principles apply because this repository is public. New repositories still stay private by default unless Ollie explicitly approves publication.

## Privacy and confidentiality
- No secrets in committed files.
- No secrets in commit messages, PR text, issue text, or code comments.
- No personal machine path leakage unless explicitly needed for user workflows.

## Simplicity and usability
- Keep README concise and human friendly.
- Put quick start near the top.
- Use simple punctuation and clear language.
- Keep advanced detail lower in the document.

## Safety and maintenance
- Keep default behaviors safe.
- Add doctor and dry-run functionality when relevant.
- Keep command outputs predictable and easy to parse.
- Prefer robust behavior over clever behavior.

## Release and visibility control
- Keep this public repository safe for public readers.
- Keep new repositories private by default.
- Publish new repositories only with explicit user consent.
- If history should remain private, use archive-and-fresh-repo workflow.
